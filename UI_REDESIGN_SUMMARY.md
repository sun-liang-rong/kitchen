## 📐 设计原则

### 1. 保持逻辑不变
所有改动仅在 UI 层，不修改任何业务逻辑、数据流和状态管理

### 2. 向后兼容
保持原有 API 接口，只新增可选参数（如 gradient, elevation）

### 3. 性能优先
- 使用 AnimatedContainer 减少重建
- SingleTickerProviderStateMixin 管理动画
- 避免不必要的 setState

### 4. 一致性
- 统一的动画时长和曲线
- 一致的间距和圆角系统
- 协调的色彩搭配
