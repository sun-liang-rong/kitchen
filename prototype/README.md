# 厨房许愿池原型说明

## 打开方式

在项目根目录运行：

```bash
cd prototype
python3 -m http.server 5173
```

然后打开：

```text
http://localhost:5173
```

也可以直接用浏览器打开 `prototype/index.html`。

## 原型包含页面

1. 许愿池首页
   - 展示双人绑定状态和邀请码。
   - 展示双方厨房状态。
   - 支持点击我的厨房状态并同步给对方。
   - 展示我许的、她许的、已认领、改天实现、先搁着等愿望卡。
   - 愿望卡展示希望时间、创建时间和最近回应。
   - 支持筛选全部、她许的、我许的、池中、待确认、已认领、改天、已兑现。

2. 许愿页
   - 输入想吃什么。
   - 选择想要的感觉。
   - 选择希望时间：今晚、明天、这周、周末、有空再说。
   - 选择愿望强度。
   - 选择是否接受替代。
   - 选择愿意搭把手的家务。
   - 提交后会在许愿池顶部新增一张愿望卡。

3. 回应愿望弹层
   - 支持今晚实现、做轻松版、换个版本、改天实现、一起做、先放池子里。
   - 支持选择改天时间和原因标签。
   - 支持给对方留一句原因或提议。
   - 发送后会更新愿望卡状态。
   - 如果选择换个版本、轻松版、改天实现，会进入“待她确认”。

4. 愿望详情页
   - 展示愿望完整信息。
   - 展示状态流转。
   - 展示回应历史。
   - 支持模拟对方确认或继续修改。

5. 我们家的菜
   - 展示常吃菜、谁会做、适合时间、难度和最近反馈。
   - 兑现时勾选加入常吃后，会把新菜插入菜库顶部。

6. 兑现记录
   - 记录愿望由谁实现、谁搭把手、饭后反馈。
   - 支持选择实现人、搭手完成情况、是否加入常吃。

## 已验证交互

- 点击悬浮按钮进入许愿页。
- 提交愿望后返回许愿池、新增愿望卡并显示提示。
- 点击“我来回应”打开回应弹层。
- 点击“发给她”关闭回应弹层、更新愿望状态并显示提示。
- 点击“等待确认”进入愿望详情页。
- 在详情页点击“她同意了”，愿望会从待确认变为已认领。
- 许愿池可以按“待确认”“已认领”等状态筛选。
- 点击“记录兑现”打开兑现弹层。
- 点击“愿望已兑现”进入兑现记录页，并新增兑现记录和常吃菜。
- 点击“我的厨房状态”可以设置状态并同步。

## 设计原则

- 用“许愿”替代“点菜”。
- 用“回应、换个版本、周末安排、先放池子里”替代“接单、拒绝”。
- 用“搭把手”替代“家务交换”。
- 不做积分、排名、强制公平统计，避免情侣关系被工具化。
